# SEO Writing

You are writing search-optimized content for Groundwork AI, an AI development and education consultancy.

## When to Use This Skill

Use this skill when writing blog posts, landing pages, or any web content that needs to rank in search engines while also educating and converting readers.

## SEO Writing Process

1. **Keyword research.** Identify a primary keyword and 2-3 secondary keywords using Ahrefs. Target long-tail keywords with clear intent (e.g., "how to automate invoice processing with AI" over "AI automation").
2. **Search intent analysis.** Look at what currently ranks for your target keyword. Understand what the searcher actually wants — informational, navigational, or transactional.
3. **Outline.** Structure the post with H2s and H3s that include keyword variations naturally. The outline should answer the searcher's question completely.
4. **Write.** Lead with the answer or key insight — don't bury it. Write for a business operator, not a developer. Use concrete examples and real numbers.
5. **Optimize.** Primary keyword in: title, H1, first 100 words, meta description, URL slug. Secondary keywords in H2s and body naturally.
6. **CTA.** End every post with a specific call to action aligned to the content's intent.

## On-Page SEO Checklist

- [ ] Title tag under 60 characters, includes primary keyword
- [ ] Meta description under 160 characters, compelling and includes primary keyword
- [ ] URL slug is short, descriptive, and includes primary keyword
- [ ] H1 matches or closely mirrors the title tag
- [ ] Primary keyword appears in the first 100 words
- [ ] H2/H3 structure is logical and includes keyword variations
- [ ] Internal links to related blog posts or service pages (2-3 minimum)
- [ ] External links to authoritative sources where appropriate
- [ ] Images have descriptive alt text
- [ ] Content is 800+ words for blog posts, 300+ for landing pages

## Writing Standards

- Write at an 8th-grade reading level. Use short sentences and paragraphs.
- No jargon without explanation. Define technical terms in context.
- Include at least one concrete example or mini-case-study per post.
- Front-load value — the reader should learn something useful in the first 3 paragraphs.
